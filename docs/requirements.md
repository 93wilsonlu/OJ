# Online Code Test / 林熙哲

## Background

現在的軟體工程師面試流程幾乎都會有 Coding Exam 的環節，你要負責設計一個線上程式測驗系統，面試者可以到系統上領取題目、上傳解答，系統會自動批改並給分，主管也可以在後台檢視面試者的成績，系統也要考慮到資安和穩定性方面的防護。

## Target Audience

面試者、面試主管、出題主管

## System Requirement

* 面試主管可以建立面試者帳密，面試者可以登入
* 出題主管可以建立題目、預期輸入與輸出（批改用）、執行時間限制
* 面試主管可以設定要給某位面試者的題目難度
* 面試者可以到系統上領取題目並上傳解答
* 系統至少支援兩種以上答題程式語言
* 系統會自動批改並給分，即時顯示成績給考生

  * 批改在後端背景執行，若考生中途關閉網頁，回來後依舊能看到成績
* 面試主管可以在後台檢視面試者的成績

## Advanced Requirement

* 面試者上傳惡意程式碼，不會影響系統或是個資外洩
* 多人同時繳交答案，批改不會塞車（或盡可能降低塞車情況）
* 面試者上傳太耗資源或時間的程式碼不會影響系統穩定性
* 系統需要易於部署在全新的 Server 上
* 避免各種面試者的作弊手段（包括但不限於抄襲、惡意程式碼、偷系統測資等等）

## Evaluation Criteria

* 30% 需求轉換與實作（e.g., 各功能的實作與 Demo）
* 10% 程式碼品質（程式碼的整潔度、可讀性與維護性。確保專案中沒有潛在的資安漏洞。）
* 25% 架構設計與可擴展性（e.g., 如何避免批改塞車、避免資安問題等等）
* 25% 系統測試與驗證（e.g., 測試覆蓋率（unit test, e2e test, integration test）；性能與穩定性測試（load test, stress test, performance test））
* 10% 運維與可靠性（e.g., 如何避免系統不穩、如何達成易於部署）

## NTU, NYCU 教學團預演建議

以下為 NTU, NYCU 教學團根據預演結果給的建議，供同學參考：

1. **評分項度應呈現於簡報 (可利用前面的作業整合)**
   題目評分項度每一項都應涵蓋在報告當中，要簡單提到或深入說明，請小組自行安排。
   * **(30%) 需求轉換與實作**：可利用寫過的 User Story、Empathy Map、AC。
   * **(25%) 架構設計與可擴展性**：不要只畫高層次架構；應呈現架構圖，系統架構圖 / Sequence Diagram 和 ER Model 是必要項。
   * **(25%) 系統測試與驗證**：應呈現測試方式、測試結果證明。
   * **(10%) 程式碼品質**：如何評估專案程式碼的健全度、可維護性與安全性。評分將依據客觀指標進行審查，審查項目包含：1. 簡報說明 2. 繳交之專案程式碼之綜合表現（經教學團確認，以此版為準）。各組可於期末簡報中檢附 SonarScanner（或 SonarQube）的檢測報告，說明自己如何評估程式碼品質。
   * **(10%) 運維與可靠性**：應呈現監控儀表板，監控哪些指標、原因自行安排。
